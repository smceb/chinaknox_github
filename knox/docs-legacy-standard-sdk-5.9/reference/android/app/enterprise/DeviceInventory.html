public boolean clearCallingLog ()

Since: API Level MDM_2.0

API to clear the logged calls from the MDM database.

Returns
  • true if call log cleared successfully, else false.
Throws
SecurityException If caller does not have required permissions
Usage
An administrator can use this API to clear the call log on MDM database (inbound and outbound). This API will not clear the device call log. The method returns false if the device has no voice call capability (e.g., a tablet). 

  EnterpriseDeviceManager edm = (EnterpriseDeviceManager) getSystemService(
                     EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
  DeviceInventory deviceInventoryPolicy = edm.getDeviceInventory();
  
  try {
     boolean result = deviceInventoryPolicy.clearCallingLog();
     if (true == result) {
         // Call log cleared
     }
  } catch(SecurityException e) {
      Log.w(TAG, "SecurityException: " + e);
  }
Permission
The use of this API requires the caller to have the "android.permission.sec.MDM_INVENTORY" permission which has a protection level of signature.

public boolean enableMMSCapture (boolean status)

Since: API Level MDM_5.0

API to enable or disable Multimedia Message Service (MMS) logging on MDM database.

Parameters
status true to enable MMS logging, false to disable it.
Returns
  • true if enable MMS capture is successful, else false.
Throws
SecurityException If caller does not have required permissions
Usage
An administrator can use this API to enable or disable multimedia Message Service (MMS) logging. The messages will be stored on a MDM database, separated from the device history. For a device managed by multiple administrators, logging is enabled if at least one administrator enables it. Logging is disabled only if all administrators disable it. 

 EnterpriseDeviceManager edm = (EnterpriseDeviceManager) getSystemService(
         EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
 DeviceInventory deviceInventoryPolicy = edm.getDeviceInventory();
 
 try {
     boolean result = deviceInventoryPolicy.enableMMSCapture(true);
     if (true == result) {
         // MMS logging enabled.
     }
 } catch (SecurityException e) {
     Log.w(TAG, "SecurityException: " + e);
 }
NOTE: This API may not affect third-party applications behavior, since MMS capture is bound to Samsung Messaging application. It's recommended to prevent the user from installing other messaging applications by using ApplicationPolicy APIs (like addAppPackageNameToBlackList(String) and addPackagesToPreventStartBlackList(List)).

public List<String> getDataCallLog (String time)

Since: API Level MDM_2.0

API to get the data call log for a specified period.

Parameters
time String representing the time in milliseconds for the desired date. Use null to retrieve the complete log
Returns
  • A list of all cellular data connections captured on device according to the given time. If the required log is empty, an empty object is returned. Each log entry appears as follows:

    "1453481330099;NORMAL;LTE;1073741824"
    "1453481511434;ROAMING;UMTS;8388608"
Throws
SecurityException If caller does not have required permissions
Usage
This method returns a list of strings with the data call log from a certain date. If null is provided, all the available log is returned. If data call logging is disabled, the log still appears but does not gather additional information.

Each log entry appears as follows: "[time (in milliseconds)];[network status (ROAMING/NORMAL)];[network type];[bytes (received/transmitted)]". The events to generate a new log entry are the following:

- either network status or type has changed
- network data activity has changed from inactive to active

In both cases, logged bytes consider only the network data received/transmitted in that entry. For accumulated number of bytes received/transmitted, please refer to getBytesReceivedNetwork() and getBytesSentNetwork(), respectively. 

  EnterpriseDeviceManager edm = (EnterpriseDeviceManager) getSystemService(
          EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
  DeviceInventory deviceInventoryPolicy = edm.getDeviceInventory();
 
  try {
      deviceInventoryPolicy.setDataCallLoggingEnabled(true);
      // The device will probably have log entries some time after enabling the policy.
      List list = deviceInventoryPolicy.getDataCallLog(Long.
              toString(calendar.getTimeInMillis()));
      if (null != list) {
          for(String log:list) {
              // Process data call log
          }
      }
  } catch(SecurityException e) {
      Log.w(TAG, "SecurityException: " + e);
  }
