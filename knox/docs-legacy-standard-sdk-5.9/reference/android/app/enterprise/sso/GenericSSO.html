Class Overview

This class provides APIs for MDM administrators to configure single sign-on on the device. It provides APIs to set/enroll a specific SSO provider, configure the information needed by the enrolled SSO provider and provide a list of applications (whitelist) that can make use of the SSO provider's service.

API to add a list of applications as whitelisted for an enrolled SSO provider.

Parameters
packages List of objects of type WhiteListPackage that contain information about packages that need to be whitelisted such as package name, signature and optional bundle containing application specific configuration details.
Returns
  • GenericSSO return status for this API are as shown below.

    SUCCESS - Successfully added whitelist packages.

    ERROR_FAIL - Error or failed case.

    ERROR_INVALID_PARAMETER - Invalid or null parameter provided to the method.

    ERROR_SSOPROVIDER_NOT_ENROLLED - No SSO provider has been enrolled in the Generic SSO framework yet.

    ERROR_NOT_SAME_MDM - The application trying to call this API is not the same that has configured the SSO service provider for the user.

Throws
SecurityException If caller does not have required permissions
Usage
An administrator can use this API to add a list of packages as ones that can use the enrolled SSO provider to obtain the authentication token. The SSO provider enrolled using enrollSSOVendor(String, String, Map) will serve as the authenticator for all the applications. 

 EnterpriseDeviceManager edm = (EnterpriseDeviceManager) getSystemService(EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
 GenericSSO genericSSO = edm.getGenericSSO(mContext);
 try {
 	List<WhiteListPackage> list = new List<WhiteListPackage>();
 	WhiteListPackage whiteListPackage1 = new WhiteListPackage(
 			"com.client.app1", "clientApp1_signature", null);
 	WhiteListPackage whiteListPackage2 = new WhiteListPackage(
 			"com.client.app2", "clientApp2_signature", null);
 	list.add(whiteListPackage1);
 	list.add(whiteListPackage2);
 	int result = genericSSO.addWhiteListPackages(list);
 	if (result < 0)
 		Log.e(TAG, "Error occurred in addWhiteListPackages");
 } catch (SecurityException e) {
 	Log.w(TAG, "Exception" + e);
 }

public int configureSSOByFile (byte[] configData)

Since: API Level MDM_5.2

This method is deprecated.
since MDM 5.8

API to push an XML configuration file, in the form of a byte array, containing information about the SSO provider and whitelisted applications to the Generic SSO framework. The API enforces SSO rules and application whitelist based on the information provided within the XML file before returning from the API call. Based on the contents of the XML file, the administrator can configure SSO provider and also whitelist applications with a single API call.

Parameters
configData The byte array of the configuration file. The configuration file is an XML file which contains three types of information namely SSO provider information, SSO parameters that are specific to the SSO provider and apply to all the whitelisted applications under the SSO provider and whitelist application specific information. The node names in the XML file are consistent with the needs of the protocol that is used by the SSO provider and can be found in GenericSSOConstants.OAuthRequestKeys or GenericSSOConstants.SAMLRequestKeys respectively. A sample XML file is shown below. 

 <?xml version="1.0" encoding="UTF-8"?>
 <configuration>
 	<ssoprovider servicepackagename="SSOPackageName" servicepackagesignature="SSOsignature">
 		<OAUTH_REQUEST_REDIRECT_URI value="https://redirect.url.com"/>
 		<OAUTH_REQUEST_CLIENT_ID value="sample-client-ID"/>
 		<OAUTH_REQUEST_RESPONSE_TYPE value="sample-response-type"/>
 		<whitelistpackage clientpackagename="clientApp1" clientpackagesignature="clientApp1Signature">
 			<OAUTH_REQUEST_NONCE value="sample-nonce-parameter-only-for-clientApp1"/>
 			<OAUTH_REQUEST_DISPLAY value="sample-display-parameter-only-for-clientApp1"/>
 			<OAUTH_REQUEST_SCOPE value="sample-scope-parameter-only-for-clientApp1">
 		</whitelistpackage>
 		<whitelistpackage clientpackagename="clientApp2" clientpackagesignature="clientApp2Signature">
 			<OAUTH_REQUEST_NONCE value="sample-nonce-parameter-only-for-clientApp2"/>
 			<OAUTH_REQUEST_DISPLAY value="sample-display-parameter-only-for-clientApp2"/>
 			<OAUTH_REQUEST_SCOPE value="sample-scope-parameter-only-for-clientApp2">
 		</whitelistpackage>
 	</ssoprovider>
 </configuration>
Returns
  • GenericSSO return status for this API are as shown below.

    SUCCESS - Successfully pushed the SSO configuration.

    ERROR_FAIL - Error or failed case.

    ERROR_INVALID_PARAMETER - Invalid or null parameters provided to the method.

    ERROR_NOT_SAME_MDM - The MDM trying to push a new SSO configuration file is not the same as the MDM administrator who is allowed to make changes to the SSO policy.

    ERROR_FILE_NOT_FOUND - The location where the SSO config file needs to be written is not found.

    ERROR_SSOAUTHENTICATOR_NOT_INSTALLED - The SSO authenticator package has not been installed on the device.

    ERROR_SSOAUTHENTICATOR_SIGNATURE_NOT_MATCHED - The SSO authenticator package has a signature that does not match the one provided as the parameter.

    ERROR_SECOND_SSOPROVIDER_REJECTED - Each user supports only one active SSO provider, so an attempt to register more than one will result in this return.

    ERROR_REMOTE_PROCESSING - The SSO authenticator package could not perform some remote operations. This error condition is also returned if the GenericSSO framework is not able to bind to the SSO authenticator package for any reason.

Throws
SecurityException If caller does not have required permissions
Usage
An administrator can use this API to push and enforce a set of configurations pertaining to the SSO provider and all the whitelisted applications in a user to the SSO framework. The byte array parameter is the XML config file which has entries corresponding to the data required by the SSO provider to carry out the authentication process. 

 EnterpriseDeviceManager edm = (EnterpriseDeviceManager) getSystemService(EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
 GenericSSO genericSSO = edm.getGenericSSO(mContext);
 try {
 	File configFile = new File("config.xml");
 	FileInputStream input = new FileInputStream(source);
 	byte ssoConfig[] = new byte[(int) source.length()];
 	input.read(ssoConfig);
 	int result = genericSSO.configureSSOByFile(ssoConfig);
 	if (result < 0)
 		Log.e(TAG, "Error occurred in unenrollSSOVendor");
 } catch (SecurityException e) {
 	Log.w(TAG, "Security Exception" + e);
 } catch (Exception e) {
 	Log.w(TAG, "Exception" + e);
 }
Precondition
SSO provider package must be installed in the particular user.

public int enrollSSOVendor (String authenticatorPackageName, String authenticatorPackageSignature, Map<String, String> ssoConfig)

Since: API Level MDM_5.2

This method is deprecated.
since MDM 5.8

API to enroll a specific SSO provider that can be used by whitelisted applications within a user or container. This API needs the MDM administrator to provide the details about the SSO service provider such as the package name and optional signature and other configuration maps. By calling this API, the administrator can enroll the SSO provider that needs to be used in the system, but cannot enforce any applications to use this SSO provider's services. To enforce applications, the MDM must later call the addWhiteListPackages(List) in order to set the list of applications that can use the SSO provider's tokens.

Parameters
authenticatorPackageName The package name of the SSO provider application.
authenticatorPackageSignature The signature of the SSO provider application. An optional parameter that maybe provided by the administrator to enforce signature level security within the generic SSO framework.
ssoConfig The configuration information which is needed by the authenticator. This information is common to all the whitelisted applications in that user. The keys needed to populate the map is obtained either from GenericSSOConstants.OAuthRequestKeys or GenericSSOConstants.SAMLRequestKeys accordingly. The values that are presented to the framework through this map are applicable for all the applications that will be whitelisted to use the enrolled SSO vendor. Specific values that need to be provided to each application can be passed in the map of the addWhiteListPackages(List) API.
Returns
  • GenericSSO return status for this API are as shown below.

    SUCCESS - Successfully enrolled the SSO provider.

    ERROR_FAIL - Error or failed case.

    ERROR_INVALID_PARAMETER - Invalid or null parameters provided to the method.

    ERROR_SECOND_SSOPROVIDER_REJECTED - Each user supports only one active SSO provider, so an attempt to register more than one will result in this return.

    ERROR_SSOAUTHENTICATOR_NOT_INSTALLED - The SSO authenticator package has not been installed on the device.

    ERROR_SSOAUTHENTICATOR_SIGNATURE_NOT_MATCHED - The SSO authenticator package has a signature that does not match the one provided as the parameter.

    ERROR_REMOTE_PROCESSING - The SSO authenticator package could not perform some remote operations. This error condition is also returned if the GenericSSO framework is not able to bind to the SSO authenticator package for any reason.

Throws
SecurityException If caller does not have required permissions
Usage
An administrator can use this API to enroll a package as an SSO provider. Once enrolled, the enrolled SSO package will serve all the whitelisted applications and help them in obtaining access tokens needed to access web resources. 

 EnterpriseDeviceManager edm = (EnterpriseDeviceManager) getSystemService(EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
 GenericSSO genericSSO = edm.getGenericSSO(mContext);
 try {
 	Map<String, String> configMap = new HashMap<String, String>();
 	configMap.put(GenricSSOConstants.OAuthRequestKeys.REDIRECT_URI,
 			"http://sample.redirect.uri");
 	configMap.put(GenricSSOConstants.OAuthRequestKeys.SCOPE, "sample_scope");
 	configMap.put(GenricSSOConstants.OAuthRequestKeys.RESPONSE_TYPE,
 			"sample_response_type");
 	int result = genericSSO.enrollSSOVendor("sso.package.name", "SSOsignature",
 			configMap);
 	if (result < 0)
 		Log.e(TAG, "Error occurred in enrollSSOVendor");
 } catch (SecurityException e) {
 	Log.w(TAG, "Exception" + e);
 }
Precondition
The SSO provider package must be installed in the particular user.
Permission
The use of this API requires the caller to have the "com.sec.enterprise.mdm.permission.MDM_SSO" permission with a protection level of signature.
