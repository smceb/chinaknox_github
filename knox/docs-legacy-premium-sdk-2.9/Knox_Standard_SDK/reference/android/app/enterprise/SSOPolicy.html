Class Overview

This class provides APIs to setup and manage Single Sign-On (SSO) services outside a container. In order to enable the SSO feature, the administrator needs to install the sso service first. Administrator needs to call below api depending on device's MDM version.

Administrator needs to call installApplication(String, boolean) with valid input parameters to install the service apk preloaded on device. Only one sso solution could be installed in a user space. So, administrator should uninstall previous SSO service app if he wants to switch SSO solution type. Only administrator must be allowed to uninstall the SSO service and user must be restricted to uninstall the app. Only one SSO service either Centrify's or Samsung's solution can be installed in a specific user space. To install Centrify SSO in USER ID say 0, first administrator checks wheather Samsung SSO is already installed in USER ID 0 or not, and if it is already installed, uninstall Samsung SSO in USER ID 0 and then install Centrify SSO and vice versa. Package path for Centrify's SSOService apk is currently defined in the variable SSO_CENTRIFY_SERVICE_PACKAGE_PATH and package path for Samsung SSO's service apk is defined in the variable SSO_SAMSUNG_SERVICE_PACKAGE_PATH

However, few models do not have preloaded sso service apk because of the limitation of ROM size. Furthermore, SSO_SAMSUNG_SERVICE_PACKAGE_PATH is deprecated since Knox 2.4 and SSO_CENTRIFY_SERVICE_PACKAGE_PATH is deprecated since Knox 2.6. Therefore, MDM Agent always has to check if the apk file is present in the device or not. If the file exists and not yet installed, MDM Agent has to install it. If the file does not exist, MDM Agent has to skip installation so that IT-admin can install this apk later.

SSO Solution Package Names
CENTRIFY_SSO_PKG : has been supported since MDM 5.0
SAMSUNG_SSO_PKG : has been supported since MDM 5.0

SSO service should be installed in a container before calling getSSOPolicy(String). 

 [How to install]
 EnterpriseDeviceManager edm = (EnterpriseDeviceManager)getSystemService(EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
 ApplicationPolicy appPolicy = edm.getApplicationPolicy();
 try {
     File file = new File(SSO_CENTRIFY_SERVICE_PACKAGE_PATH or SSO_SAMSUNG_SERVICE_PACKAGE_PATH);
     if(file.exists() && !appPolicy.isApplicationInstalled(CENTRIFY_SSO_PKG or SAMSUNG_SSO_PKG)) {
         appPolicy.installApplication(SSO_CENTRIFY_SERVICE_PACKAGE_PATH or SSO_SAMSUNG_SERVICE_PACKAGE_PATH, false);
     }
 } catch (SecurityException e) {
     Log.w(TAG, "SecurityException : " + e);
 }
 edm.getSSOPolicy()
 

 [How to uninstall]
 try {
     appPolicy.uninstallApplication(CENTRIFY_SSO_PKG or SAMSUNG_SSO_PKG, false);
 } catch (SecurityException e) {
     Log.w(TAG, "SecurityException: " +  e);
 }
 

 [How to get SSO instance]
 EnterpriseDeviceManager edm = (EnterpriseDeviceManager)getSystemService(EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
 SSOPolicy mSSOPolicy = edm.getSSOPolicy(CENTRIFY_SSO_PKG or SAMSUNG_SSO_PKG);

The method isSSOReadyInUserSpace(String) should be called before calling other APIs such as deleteSSOWhiteListInUserSpace(String, List), getAppAllowedStateInUserSpace(String, String), pushSSODataInUserSpace(String, Bundle), setCustomerInfoInUserSpace(String, String, String), setSSOWhiteListInUserSpace(String, List), unenrollInUserSpace(String) to check for service bind status. If isSSOReadyInUserSpace(String) returns false, setupSSOInUserSpace(String) should be called before calling the above mentioned APIs.

 EnterpriseDeviceManager edm = (EnterpriseDeviceManager)getSystemService(EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);

 if(!mSSOPolicy.isSSOReadyInUserSpace(CENTRIFY_SSO_PKG or SAMSUNG_SSO_PKG)){
     if(mSSOPolicy.setupSSOInUserSpace(CENTRIFY_SSO_PKG or SAMSUNG_SSO_PKG) == SSOPolicy.SSO_RESULT_FAILURE){
         return;
     }        
 }

 try {
     Bundle bundle = new Bundle();

     StringBuffer MDMRevisedIDPUrl = null;

SecurityException If caller does not have required permissions
UnsupportedOperationException The API is not supported by the SSO type.
Usage

The specified SSO solution should be installed in userspace before calling this api. This API should be called for requesting platform SSOPolicy service to bind to an SSO solution. 

 try {
     mSSOPolicy.setupSSOInUserSpace(CENTRIFY_SSO_PKG);
 } catch (SecurityException e) {
     Log.w(TAG, "Exception" + e);
 }
