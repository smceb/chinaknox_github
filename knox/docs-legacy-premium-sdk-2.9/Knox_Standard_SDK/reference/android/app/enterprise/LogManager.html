Class Overview

This class is to be used only for debugging and can be used only by authorized administrator applications. This class provides APIs to control internal logging of the enterprise device management framework.

There are two types of logging facilities provided: Console logging and file logging. Both types can be active simultaneously. By default, console logging is enabled and file logging is disabled. Once any type of logging (console or file) is enabled, it is persistent across reboots.

The sensitive level LVL_SENSITIVE is never logged to the console log. Logging at the sensitive level is only to the file log. The logs at this level are expected to contain sensitive information, hence logging them to the console log could potentially lead to leakage of sensitive information to any application or user via the 'logcat' command of the adb shell.

In case file logging is enabled, the administrator can retrieve the stored log file. Logs are stored for a maximum of 5 days from the day they are logged. All older logs are deleted. For example, logs logged on 1st June 2012 are deleted at the end of the day on 6th June 2012.

public static int copyLogs (ParcelFileDescriptor file)

Since: API Level MDM_4.0

API to copy stored log file of the enterprise device management framework to a specified file provided by the administrator.

Parameters
file The parcelable file descriptor of the file where the logs should be copied to. The file should be writable.
Returns
  • One of ERROR_NONE,ERROR_UNKNOWN, ERROR_UNAUTHORIZED, ERROR_IO_WRITE, or ERROR_LOG_TYPE_NOT_ACTIVE (if file logging is not enabled)
Throws
SecurityException If caller is not a registered authorized administrator on the device
Usage
An administrator can use this API to copy the log file maintained by the enterprise device management framework to any specified file. This API works only if file logging has been enabled already. The ParcelFileDescriptor object passed should be a file and should be opened with write permissions. 

 File file = new File(logPath);
 
 try {
 	ParcelFileDescriptor pfd = ParcelFileDescriptor.open(file,
 			ParcelFileDescriptor.MODE_READ_WRITE
 					| ParcelFileDescriptor.MODE_CREATE
 					| ParcelFileDescriptor.MODE_TRUNCATE);
 	int result = LogManager.copyLogs(pfd);
 } catch (FileNotFoundException e) {
 	e.printStackTrace();
 }

public static int getLogLevel (int type)

Since: API Level MDM_4.0

API to get the log level that is currently used for a specific type of logging in the enterprise device management framework.

Parameters
type The type of logging for which the log level is to be queried. One of LOG_TYPE_CONSOLE or LOG_TYPE_FILE.
Returns
  • The log level currently used for logging. Can be one of LVL_SENSITIVE, LVL_VERBOSE, LVL_DEBUG, LVL_INFO, LVL_WARNING, or LVL_ERROR. On failure one of ERROR_INVALID_LOG_TYPE, ERROR_LOG_TYPE_NOT_ACTIVE, or ERROR_UNKNOWN.
Throws
SecurityException If caller is not a registered authorized administrator on the device
Usage
An administrator can use this API to get the log level currently used for either file logging or console logging. All log levels above and including the given log level are logged. 

 try {
 	if (LogManager.isLoggingEnabled(LogManager.LOG_TYPE_CONSOLE)) {
 		int logLevel = LogManager.getLogLevel(LogManager.LOG_TYPE_CONSOLE);
 	}
 } catch (SecurityException e) {
 	Log.w(TAG, "SecurityException: " + e);
 }

public static int setLogLevel (int type, int level)

Since: API Level MDM_4.0

API to set the log level to be used for a specific type of logging in the enterprise device management framework.

Parameters
type The type of logging for which the log level is to be set. One of LOG_TYPE_CONSOLE or LOG_TYPE_FILE.
level The log level to be used for logging. The levels can be one of LVL_SENSITIVE, LVL_VERBOSE, LVL_DEBUG, LVL_INFO, LVL_WARNING, or LVL_ERROR.
Returns
  • One of ERROR_NONE,ERROR_UNKNOWN, ERROR_UNAUTHORIZED, ERROR_INVALID_LOG_TYPE, ERROR_INVALID_LOG_LEVEL, or ERROR_LOG_TYPE_NOT_ACTIVE
Throws
SecurityException If caller is not a registered authorized administrator on the device
Usage
An administrator can use this API to set the log level to be used for either file logging or console logging. All log levels above and including the given log level are logged. Make sure logging is enabled for a specific type before changing the log level. 

 try {
 	if (LogManager.isLoggingEnabled(LogManager.LOG_TYPE_CONSOLE)) {
 		LogManager.setLogLevel(LogManager.LOG_TYPE_CONSOLE,
 				LogManager.LVL_WARNING);
 	}
 } catch (SecurityException e) {
 	Log.w(TAG, "SecurityException: " + e);
 }
