Class Overview

Public interface for managing policies enforced on a device. Most clients of this class must have published a DeviceAdminReceiver that the user has currently enabled.

Enterprise Migration


Enterprise migration is the process of upgrading the enterprise policies from a lower version to a higher version. Registering for ACTION_ENTERPRISE_MIGRATION intent is always safe. This intent is broadcasted when any enterprise framework upgrade happens from MDM v1.0 to any higher version. The extra EXTRA_MIGRATION_RESULT provides the status of the migration as a boolean. If true, the migration completed successfully, else the migration failed, and the administrator client must query the policies applied and re-apply if required.

Note: When migration from MDM v1.0 to any higher version occurs and there is more than one administrator registered on the device, the migration fails, and the administrator must re-apply the policies. Any subsequent migration from MDM v2.1 to any higher version does not fail, and hence the ACTION_ENTERPRISE_MIGRATION intent is not broadcasted.

public static final int PASSWORD_QUALITY_SMARTCARDNUMERIC

Since: API Level MDM_4.0

This constant is deprecated.
Use PASSWORD_QUALITY_SMARTCARDNUMERIC

Constant for setPasswordQuality(ComponentName,int). The user must have entered a password containing all numeric characters which is used to authenticate user with SmartCard.

To set SmartCard based device lock screen, user should have SmartCard,SmartCard reader and also have necessary Smartcard drivers and applications installed on the device. Administrator can enforce SmartCard based device lock screen by requesting device password quality as PASSWORD_QUALITY_SMARTCARDNUMERIC with setPasswordQuality(ComponentName,int) and enforce password change with enforcePwdChange(). PASSWORD_QUALITY_SMARTCARDNUMERIC quality is higher value than PASSWORD_QUALITY_COMPLEX. 

 EnterpriseDeviceManager edm = (EnterpriseDeviceManager) getSystemService
      (EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
 DevicePolicyManager dpm = (DevicePolicyManager) getSystemService(Context.DEVICE_POLICY_SERVICE);
 PasswordPolicy passwordPolicy = edm.getPasswordPolicy();
 
 try {
     // Set password quality 
     dpm.setPasswordQuality(enterpriseDeviceAdmin, EnterpriseDeviceManager.PASSWORD_QUALITY_SMARTCARNUMERIC);
     //set minimum length of password. Each smartcard may have different length configured as password. So it depends upon the SmartCard type
     dpm.setPasswordMinimumLength(enterpriseDeviceAdmin,8)
     boolean result = passwordPolicy.enforcePwdChange();
     if (true == result) {
         // if successful user required to create a smartcard password with no
         // option to cancel dialog.
     }catch (SecurityException e) {
     Log.w(TAG, "SecurityException: " + e);
 }

public int selfUpdateAdmin (String apkFilePath)

Since: API Level MDM_3.0

This method is deprecated.
Not supported

API to update the calling administrator application.

This API updates the administrator application that calls this API. The three main requirements for this update to succeed are :
1). The new apk passed via the apkFilePath parameter must have the same package name as the calling administrator.
2). The administrator component in the new apk must be the same component as in the old apk.
3). The calling administrator application must be an active device administrator when this API is called.
4). The new apk must either have a valid payload or be signed with the platform key of the device on which the call is made.

This API enables an older version of the administrator application to update to a newer version even though the older and newer versions are signed with different keys.

This update occurs without user interaction.

NOTE: USE THIS API ONLY AS A LAST RESORT. All applied policies are removed. The administrator must re-apply all policies.

Parameters
apkFilePath Path of the new apk in the file system (e.g., /data/example.apk)
Returns
  • ERROR_NONE,ERROR_INVALID_FILE, ERROR_PACKAGE_NAME_MISMATCH, ERROR_CRYPTO_CHECK_FAILURE, ERROR_NOT_ACTIVE_ADMIN

public boolean setAdminRemovable (boolean removable, String packageName)

Since: API Level MDM_4.0

API to set the removable status of another administrator application.

Parameters
removable true enables administrator removal, false disables administrator removal
packageName Name of administrator whose removable status is set
Returns
  • true on success, false on failure
Usage
An administrator can set the removable status of another administrator application. If set to true, the user can remove the administrator through the Settings application. If set to false, the user cannot remove the administrator through the Settings application.

Note: Currently this API does not support rollback so it is administrator responsibility to revoke this policy before de-registering itself.

Permission
The use of this API requires the caller to have the "android.permission.sec.ENTERPRISE_DEVICE_ADMIN" permission which has a protection level of signature.
