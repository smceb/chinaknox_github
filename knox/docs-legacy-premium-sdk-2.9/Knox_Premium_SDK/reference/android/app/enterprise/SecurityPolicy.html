public boolean addPackagesToCertificateWhiteList (List<AppIdentity> packageList)

Since: API Level

API to configure packages without MDM permissions to manage certificates in the device.

Parameters
packageList List of AppIdentity containing applications that shall be able to call MDM APIs to manage certificates.
Returns
  • True if the list of applications have been successfully added to the whitelist, false if at least one application has failed to be added to the whitelist.
Throws
SecurityException If caller does not have required permissions
Usage
An administrator can use this API to provide permissions to a trusted application to use certificate management APIs without being an active admin nor having MDM permissions.
A package whitelisted with this policy will be able to call the set of APIs listed below.

Standard SDK:

  • installCertificateToKeystore(String, byte[], String, String, int)
  • getCertificatesFromKeystore(int)
  • deleteCertificateFromKeystore(CertificateInfo, int)
  • getCredentialStorageStatus()
Premium SDK:

Please refer to Certificate Enrollment APIs in com.sec.enterprise.knox.certenroll package. The whitelisted application will be able to do certificate enrollment under SCEP and EST protocol without any MDM permissions. As the APIs related to this feature are only available in Premium SDK, the administrator that has whitelisted the application must have been activated with KLM license. Otherwise, the whitelisted application will not be able to use such feature. 

 EnterpriseDeviceManager edm = (EnterpriseDeviceManager) getSystemService(
         EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
 SecurityPolicy securityPolicy = edm.getSecurityPolicy();

 try {
     // In order to validate the application's signature, fill this variable with the desired
     // signature.
     String signature = null;
 
     // List of packages that shall be able to use certificate management APIs
     List<AppIdentity> packageList = new ArrayList<AppIdentity>();
     packageList.add(new AppIdentity("com.trusted.app", signature));
 
     boolean result = securityPolicy.addCertificateDelegationPackages(packageList);
     if (result == true) {
         // List of packages successfully added to certificate management whitelist
     }
 } catch (SecurityException e) {
     Log.w(TAG, "SecurityException: " + e);
 }

public boolean deleteCertificateFromKeystore (CertificateInfo certInfo, int keystore)

Since: API Level

API to remove or disable a certificate from the given keystore(s).

Parameters
certInfo Certificate wrapper containing the certificate object (X509Certificate) to be removed from keystore(s).
keystore Keystore(s) from where certificates must be removed or disabled. Possible values are KEYSTORE_DEFAULT, KEYSTORE_FOR_WIFI and KEYSTORE_FOR_VPN_AND_APPS if caller is on owner scope. If caller is in an user, KEYSTORE_FOR_WIFI keystore cannot be used, otherwise the API will throw a SecurityException. These values are bitwise and can be combined with logical OR.
Returns
  • true if certificate was removed from keystore(s) successfully, false otherwise.
Throws
SecurityException If caller does not have required permissions or caller is in a guest user or container trying to iterate with a global keystore, e.g. KEYSTORE_FOR_WIFI)
Usage
An administrator can use this API to silently uninstall a certificate from certain keystore(s) or disable system pre-loaded ones (as this kind of certificate cannot be deleted). 

  EnterpriseDeviceManager edm = (EnterpriseDeviceManager) getSystemService(
      EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
  SecurityPolicy securityPolicy = edm.getSecurityPolicy();

  String filePath = "/sdcard/class2.crt";
  byte[] certData = getByteArray(filePath); //internal function to retrieve
                                           //byte array from file.

  try {
      boolean result1, result2;
      CertificateFactory certFactory = CertificateFactory.getInstance("X.509");
      X509Certificate cert = (X509Certificate) certFactory
              .generateCertificate(new ByteArrayInputStream(certData));

      CertificateInfo certInfo = new CertificateInfo(cert);

      // uninstalling certificate from Android Default keystore
      // if certInfo is a system certificate, then it will only be disabled in system.
      result1 = securityPolicy.deleteCertificateFromKeystore(certInfo, 
               SecurityPolicy.KEYSTORE_DEFAULT);

      if(SecurityPolicy.KEYSTORE_NO_ERROR ==
           securityPolicy.getCredentialStorageStatus()) {
           // uninstalling certificate from Wifi and VPN and Apps keystores (it will only work for owner space)
           result2 = securityPolicy.deleteCertificateFromKeystore(certInfo,
              SecurityPolicy.KEYSTORE_FOR_WIFI | SecurityPolicy.KEYSTORE_FOR_VPN_AND_APPS);
      }

      if(result1 == true) {
          // certificate successfully removed/disabled from Android Default keystore
      }

      if(result2 == true) {
          // certificate successfully removed from WiFi and VPN and Apps keystores
      }
  } catch(SecurityException e) {
      Log.w(TAG, "SecurityException: "+ e);
  } catch (CertificateException e) {
      Log.w(TAG, "CertificateException: " + e);
  }
For Container:
  // When you create container successfully, containerID will be returned via intent.
  // Use this containerID in below API.
  EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
  KnoxContainerManager kmcm = ekm.getKnoxContainerManager(mContext, containerID);
  SecurityPolicy securityPolicy = kmcm.getSecurityPolicy();

  String filePath = "/sdcard/class2.crt";
  byte[] certData = getByteArray(filePath); //internal function to retrieve
                                           //byte array from file.

  try {
      boolean result = false;
      CertificateFactory certFactory = CertificateFactory.getInstance("X.509");
      X509Certificate cert = (X509Certificate) certFactory
              .generateCertificate(new ByteArrayInputStream(certData));

      CertificateInfo certInfo = new CertificateInfo(cert);

      // uninstalling certificate from VPN and applications keystore
      result = securityPolicy.deleteCertificateFromKeystore(certInfo,
              SecurityPolicy.KEYSTORE_FOR_VPN_AND_APPS);

      if(result == true) {
          // certificate successfully removed from VPN and Apps keystore of container
      }
  } catch(SecurityException e) {
      Log.w(TAG, "SecurityException: "+ e);
  } catch (CertificateException e) {
      Log.w(TAG, "CertificateException: " + e);
  }
NOTE: This API will return false if KEYSTORE_DEFAULT is used as keystore parameter and container is in locked state.

Permission

public List<CertificateInfo> getCertificatesFromKeystore (int keystore)

Since: API Level

API to get the list of certificates kept by the given keystore(s).

Parameters
keystore Keystore(s) from where certificates must be retrieved. Possible values are KEYSTORE_DEFAULT, KEYSTORE_FOR_WIFI, and KEYSTORE_FOR_VPN_AND_APPS if caller is on owner scope. If caller is in an user, KEYSTORE_FOR_WIFI keystore cannot be used, otherwise the API will throw a SecurityException. These values are bitwise and can be combined with logical OR.
Returns
  • list of certificates kept by the given keystore(s) or an empty list if none are found or null if some error occurred.
Throws
SecurityException If caller does not have required permissions or caller is in a guest user or container trying to iterate with a global keystore, e.g. KEYSTORE_FOR_WIFI)
Usage
An administrator can use this API to get a list containing all certificates stored in certain keystore(s). This API returns a list containing CertificateInfo objects, which are wrappers to Certificate type objects; cast to X509Certificate in order to read them in a detailed manner. 

  EnterpriseDeviceManager edm = (EnterpriseDeviceManager) getSystemService(
      EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
  SecurityPolicy securityPolicy = edm.getSecurityPolicy();

  try {

     //retrieving certificates from all keystores (this call will work only on owner space):
     List<CertificateInfo> list = securityPolicy.getCertificatesFromKeystore
             (SecurityPolicy.KEYSTORE_DEFAULT |
             SecurityPolicy.KEYSTORE_FOR_WIFI |
             SecurityPolicy.KEYSTORE_FOR_VPN_AND_APPS);

     if (list != null && !list.isEmpty()) {
         X509Certificate cert;

         for(CertificateInfo certInfo : list) {
             Log.d(TAG, "Preloaded : " + certInfo.getSystemPreloaded()
                     + ". Enabled : " + certInfo.getEnabled());

             cert = (X509Certificate)certInfo.getCertificate();
             Log.d(TAG, "Version :" + cert.getVersion()
                     + ". Serial no.: " + cert.getSerialNumber().toString());
         }
     }
  } catch(SecurityException e) {
     Log.w(TAG,"SecurityException: "+e);
  }
For Container:
  // When you create container successfully, containerID will be returned via intent.
  // Use this containerID in below API.
  EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
  KnoxContainerManager kmcm = ekm.getKnoxContainerManager(mContext, containerID);
  SecurityPolicy securityPolicy = kmcm.getSecurityPolicy();

  try {

     // retrieving certificates from VPN and Apps keystore
     List<CertificateInfo> list = securityPolicy.getCertificatesFromKeystore
             (SecurityPolicy.KEYSTORE_FOR_VPN_AND_APPS);

     if (list != null && !list.isEmpty()) {
         X509Certificate cert;

         for(CertificateInfo certInfo : list) {
             cert = (X509Certificate)certInfo.getCertificate();
             Log.d(TAG, "Version :" + cert.getVersion()
                     + ". Serial no.: " + cert.getSerialNumber().toString());
         }
     }
  } catch(SecurityException e) {
     Log.w(TAG,"SecurityException: "+e);
  }
NOTE: This API will return an empty list if KEYSTORE_DEFAULT is used as keystore parameter and container is in locked state.

public int getCredentialStorageStatus ()

Since: API Level

API to get the current credential storage (Default, Wifi and VPN and applications keystores) state.

Returns
  • Keystore error code:

    KEYSTORE_NO_ERROR if ready to use and is unlocked

    KEYSTORE_UNINITIALIZED if password not yet set.

    KEYSTORE_LOCKED if credentials storage is locked

Throws
SecurityException If caller does not have required permissions

The use of this API requires the caller to have the "android.permission.sec.MDM_SECURITY" permission which has a protection level of signature.
Usage
An administrator can use this API to check whether a password has already been set to a credential storage, thus if it is ready to be used or not. Certificates can only be installed, listed or deleted from Default, Wifi or VPN and applications keystores when credential storage is in KEYSTORE_NO_ERROR state.

public boolean installCertificateToKeystore (String type, byte[] value, String name, String password, int keystore)

Since: API Level

API to install a certificate into the given keystore(s) silently.

Parameters
type TYPE_CERTIFICATE or TYPE_PKCS12.
value A non-empty byte array containing the certificate data.
name A non-empty String with the alias to identify the certificate in KeyStore.
password A non-null String with the password to decrypt the input
keystore Target keystore for certificate installation. Possible values are KEYSTORE_DEFAULT, KEYSTORE_FOR_WIFI and KEYSTORE_FOR_VPN_AND_APPS if caller is on owner scope. If caller is in an user, KEYSTORE_FOR_WIFI keystore cannot be used, otherwise the API will throw a SecurityException. These values are bitwise and can be combined with logical OR.
Returns
  • true if certificate is installed successfully to all selected keystores, false if it failed installing to at least one keystore (even if it successfully installed on other keystores).
Throws
SecurityException If caller does not have required permissions or caller is in a guest user or container trying to iterate with a global keystore, e.g. KEYSTORE_FOR_WIFI)
Usage
An administrator can use this API to install certificates of type TYPE_CERTIFICATE or TYPE_PKCS12 silently without user interaction. Possible target keystores are KEYSTORE_DEFAULT, KEYSTORE_FOR_WIFI and KEYSTORE_FOR_VPN_AND_APPS when caller is on owner space. If caller is in an user, KEYSTORE_FOR_WIFI keystore cannot be used, otherwise the API will throw a SecurityException. The credential storage must be in KEYSTORE_NO_ERROR state (screen lock set with a Password or PIN), otherwise this API will fail. In order to get the credential storage state, please consider using getCredentialStorageStatus() API. 

  EnterpriseDeviceManager edm = (EnterpriseDeviceManager) getSystemService(
      EnterpriseDeviceManager.ENTERPRISE_POLICY_SERVICE);
  SecurityPolicy securityPolicy = edm.getSecurityPolicy();

  String filePath = "/sdcard/class3.crt";
  byte[] certData = getByteArray(filePath); //internal function to retrieve
                                           //byte array from file.
  try {
      boolean result1, result2;

      //installing into Wifi keystore (it will only work for owner space):
      result1 = securityPolicy.installCertificateToKeystore
              (SecurityPolicy.TYPE_CERTIFICATE, certData, "mycert", "123456",
              SecurityPolicy.KEYSTORE_FOR_WIFI);

      //installing into Wifi and VPN and Apps keystores (it will only work for owner space):
      if(SecurityPolicy.KEYSTORE_NO_ERROR ==
           securityPolicy.getCredentialStorageStatus()) {
           result2 = securityPolicy.installCertificateToKeystore
               (SecurityPolicy.TYPE_CERTIFICATE, certData, "mycert", "123456",
               SecurityPolicy.KEYSTORE_FOR_WIFI | SecurityPolicy.KEYSTORE_FOR_VPN_AND_APPS);
      }

      if(true == result1) {
          //certificate installed to Wifi keystore successfully
      }

      if(true == result2) {
          //certificate installed to Wifi and VPN and Apps keystores successfully
      }
  } catch(SecurityException e) {
      Log.w(TAG,"SecurityException: "+e);
  }
For Container:
  // When you create container successfully, containerID will be returned via intent.
  // Use this containerID in below API.
  EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();

  }
