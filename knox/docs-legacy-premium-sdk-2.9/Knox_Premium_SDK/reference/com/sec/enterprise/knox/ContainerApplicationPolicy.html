This method is deprecated.
Use addPackageToInstallWhiteList(String)

API to be used by Container administrator to add the package name of a containerized application in to a white list. By adding the package name in the white list Container administrator gives the permission to that application to install other applications inside the container. Some containerized apps if white listed by an Container administrator, can access Container serivce to install other application inside the container.

Parameters
packageName The package name of the application to be added to the white list. (e.g., com.abc.xyz)
Returns
  • true if application package added successfully to the white list, else false.
Throws
SecurityException If caller does not have required permissions. Only administrator who created container can enforce policy, if the calling administrator is not an owner of container the API throws SecurityException.
IllegalArgumentException If container does not exist or container creation/removal is in progress.
Usage
An administrator can use this API to add a containerized application in the white list in order to give access to install other applications inside the container. This API basically adds the package name in the white listed applications list.
addPackageToInstallWhiteList(String). 

 // When you create container successfully, containerID will be returned via intent.
 // Use this containerID in below API.
 EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
 try {
     EnterpriseContainerManager containerMgr = ekm.getEnterpriseContainerManager(containerID);
     ContainerApplicationPolicy cap = containerMgr.getContainerApplicationPolicy();

     if (true == result) {
         Log.d(TAG, "Package added to install white list successfully");
     } else {
         Log.d(TAG, "Failed to add Package to install white list.");
     }

     Log.w(TAG, "SecurityException: " + e);

public boolean addPackagesToClearCacheBlackList (List<String> packageList)

Since: API Level KNOX_1.0

This method is deprecated.
Use addPackagesToClearCacheBlackList(List)

API to add a list of packages to the clear cache blacklist.

Parameters
packageList List of packages to be added to the clear cache blacklist.
Returns
  • true if the package list was successfully added, else false.
Throws
SecurityException If caller does not have required permissions. Only administrator who created container can enforce policy, if the calling administrator is not an owner of container the API throws SecurityException.
IllegalArgumentException If container does not exist or container creation/removal is in progress.
Usage

An administrator can use this API to prevent the user from clearing cache for certain applications. The actions include clear cache in Settings app, clearing cache through third- party applications, and clearing service cache from the application. The user can still clear cache the application through its interface or back key. The administrator provides a list of packages that are added to the blacklist. For any package added on the whitelist using addPackagesToClearCacheWhiteList(List) , then that application cache can be cleared. The whitelist is the exception to the blacklist. If the package name matches the pattern in both the blacklist and whitelist, the whitelist takes priority.

Package name specifications containing Wildcard characters (e.g., "com.xyz.*") are allowed. A Wildcard character is a character that may be substituted for any of a defined subset of all possible characters. The asterisk character ("*") substitutes for any zero or more characters, and the question mark ("?") substitutes for any one character or less but not more than the amount of question marks. 

 List<String> list = new ArrayList<String>();
 list.add("com.android.settings");
 list.add("com.rovio.angrybirdsrio");
 // When you create container successfully, containerID will be returned via intent.
 // Use this containerID in below API.
 EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
 try {
     EnterpriseContainerManager containerMgr = ekm.getEnterpriseContainerManager(containerID);
     ContainerApplicationPolicy cap = containerMgr.getContainerApplicationPolicy();
     boolean result = cap.addPackagesToClearCacheBlackList(list);
     if (true == result) {
         // user is not allowed to clear cache the blacklisted application
         // packages
     } else {
         // previous behaviour for device still prevails
     }
 } catch (SecurityException e) {
     Log.w(TAG, "SecurityException: " + e);
 }
Permission
The use of this API requires the caller to have the "android.permission.sec.MDM_APP_MGMT" permission which has a protection level of signature.
