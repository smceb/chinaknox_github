public boolean addNewAdminActivationAppWhiteList (List<String> packageNames)

Since: API Level KNOX_2.0

API to add list of whitelisted applications that can be activated as an administrator.

Returns
  • true if whitelisted administrator activated applications is added, false if not added
Usage
An administrator can use this API to add list of whitelisted applications that are allowed to be activated as an administrator. 

 EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
 
 try {
     List packageList = new ArrayList();
     packageList.add("com.android.email");

     if (ekm.getAdvancedRestrictionPolicy().addNewAdminActivationAppWhiteList(packageList)) {
         Log.w(TAG, " Add new administrator activation whitelist allowed");
     } else {
         Log.w(TAG, "Failed to set the policy.");
     }
 } catch (SecurityException e) {
     Log.w(TAG, "Exception" + e);
 }
For Container:
 // When you create container successfully, containerID will be returned via intent.
 // Use this containerID in below API.
 EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
 KnoxContainerManager kmcm = ekm.getKnoxContainerManager(mContext, containerID);
 AdvancedRestrictionPolicy advancedRestrictionPolicy = kmcm.getAdvancedRestrictionPolicy();
 try {
     List packageList = new ArrayList();
     packageList.add("com.android.email");
     if (advancedRestrictionPolicy.addNewAdminActivationAppWhiteList(packageList)) {
         Log.w(TAG, " Add administrator activation whitelist allowed");
     } else {
         Log.w(TAG, "Failed to set the policy.");
     }
 } catch (SecurityException e) {
     Log.w(TAG, "Exception" + e);
 }
Permission
The use of this API requires the caller to have the "com.sec.enterprise.knox.permission.KNOX_RESTRICTION" permission with a protection level of signature.

public boolean setCCMode (boolean enable)

Since: API Level KNOX_2.0

API to enable or disable CC (Common Criteria) mode.

Parameters
enable true to enable, false to disable.
Returns
  • true if operation is successful, else false.
Throws
SecurityException If caller does not have required permissions
Usage
An administrator can enable the device to be placed into the Common Criteria configuration. When enabled, bootloader will block KIES download mode, enforce to check integrity of kernel, and self-test crypto modules. In addition, device will verify additional signature on FOTA update using RSA-PSS signature and enforce to use FIPS 140-2 validated crypto module for EAP-TLS Wi-Fi connection.
To fully enable Common Criteria-evaluated configuration, below prerequisites should be satisfied :
1) Enable Device Encryption : Administrator must enable device encryption by using API android.app.enterprise.SecurityPolicy.setRequireDeviceEncryption(ComponentName, boolean). Before calling device encryption API, Administrator needs to disable fast encryption using android.app.enterprise.RestrictionPolicy.allowFastEncryption(boolean).
2) Enable SD Card Encryption : Administrator must enable external storage encryption by using API android.app.enterprise.SecurityPolicy.setRequireStorageCardEncryption(ComponentName, boolean)
3) Set maximum Password Attempts before Wipe : Administrator must set maximum failed password attempts for wipe by using API DevicePolicyManager.setMaximumFailedPasswordsForWipe(ComponentName, int)
4) Enable Certificate Revocation (Since KNOX 2.2) : Administrator must enable certificate revocation by using API enableRevocationCheck(String, boolean)
5) Disable Password History (Since KNOX 2.2) : Administrator must disable password history by using API setPasswordHistory(ComponentName, int)

Note:

Before applying CC policy, the CC mode state will be "Ready". Once policy is applied, the state will change to "Enforced" until device meets all the prerequisites.
If device meets all prerequisites, CC mode will be enabled after rebooting and state will change to "Enabled".
If the device is tampered or FIPs self-test is failed, the state will change to "Disabled".
As enabling CC mode works asynchronously, return value of setCCMode(boolean) doesn't mean if CC mode is properly set or not.
To check the status of CCMode, use getCCModeState().

 EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
 try {
     boolean enable = false;
     ekm.getAdvancedRestrictionPolicy().setCCMode(enable);
     Log.w(TAG, "Set CCMode is called.");
 } catch (SecurityException e) {
     Log.w(TAG, "Exception" + e);
 }

The use of this API requires the caller to have the "com.sec.enterprise.knox.permission.KNOX_RESTRICTION" permission with a protection level of signature.
