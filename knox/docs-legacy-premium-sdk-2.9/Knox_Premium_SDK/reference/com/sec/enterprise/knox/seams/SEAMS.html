public int activateDomain ()

Since: API Level KNOX_2.0

This method is deprecated.
Since KNOX 2.7

API to activate SEAndroid protection for third party container. In order to use this API, the MDM should work with Samsung to define the requirements for the SEAndroid domain based protection. Once the requirements are established, Samsung will implement the feature for the MDM. A FOTA upgrade will be required to use this feature once development is complete.

Returns
  • Returns containerID if successful, POLICY_FAILED if method error, POLICY_REFUSED if authentication fails.
Precondition
getActivationStatus() should return false
Post result
Once activation is complete, getActivationStatus() should return true to reflect the successful activation. The calling MDM agent, once re-launched, will run in the the designated SE for android domain, and will be eligible to call other SEAMS APIs.
Usage

 SEAMS seams = SEAMS.getInstance(context);
 try {
     // activate domain
     seams.activateDomain ();
 
 } catch (SecurityException e) {
     Log.e(TAG, "SecurityException: " + e);
 }
Permission
The use of this API requires the caller to have the "com.sec.enterprise.knox.permission.KNOX_SEAMS" permission, which has a protection level of signature.
Since
KNOX 2.0
Deprecated since
KNOX 2.7
Multiuser Environment

public int addAppToContainer (String packageName, String[] certificate, int containerID, int appType)

Since: API Level KNOX_2.0

API to add an application to a specific container domain, corresponding to the appType.

Parameters
packageName - packageName of the application
certificate - Signature array based on the certificate array for the application see getSignatureFromPackage(String) or getSignatureFromCertificate(byte[]) for creating signature array in correct format
containerID - id for the container
appType - describes the domain category to which the application is being added. appType for SECURED_APPTYPE = 1. appType for TRUSTED_APPTYPE = 2. appType for GENERIC_SECURED_APPTYPE = 3. appType for GENERIC_TRUSTED_APPTYPE = 4.
Returns
  • Returns POLICY_OK if successful, POLICY_FAILED if method error, and POLICY_REFUSED if authentication fails. ERROR_NOT_SUPPORTED if the application isn't allowed to be added to container. ERROR_ALREADY_CONTAINER_APP if the application is already in a container. ERROR_CERTS_NOT_MATCHED if the input certificate does not match. ERROR_CONTAINER_ID_MISMATCH if the input Container ID is not valid. ERROR_NO_CERTS if the package is not installed and input certificate is null.
Usage

 SEAMS seams = SEAMS.getInstance(context);
 try {
     // add app to container
     seams.addAppToContainer (packageName, certificate, containerID, appType);
 
 } catch (SecurityException e) {
     Log.e(TAG, "SecurityException: " + e);
 }
Permission
The use of this API requires the caller to have the "com.sec.enterprise.knox.permission.KNOX_SEAMS" permission, which has a protection level of signature.

KNOX 2.0

In order to use this API, the MDM should work with Samsung to define the requirements for the SEAndroid domain based protection. Once the requirements are established, Samsung will implement the feature for the MDM. A FOTA upgrade will be required to use this feature once development is complete.

Returns
  • Returns POLICY_OK if successful, POLICY_FAILED if method error, and ERROR_CONTAINER_NOT_EMPTY if container is not empty, POLICY_REFUSED if authentication fails.
Precondition

Post result
The agent application that was assigned the domain based on current container, will run with the "untrusted_app" domain when restarted. getActivationStatus() returns false.
Usage

 SEAMS seams = SEAMS.getInstance(context);
 try {

     seams.deActivateDomain();
