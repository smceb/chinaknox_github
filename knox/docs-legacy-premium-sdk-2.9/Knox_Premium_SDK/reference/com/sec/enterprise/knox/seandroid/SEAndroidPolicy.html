Class Overview

This class provides APIs to manage policy rules into the SEAndroid. After applying any SE Android policy on device, we recommend device reboot. Administrator can call safe api PasswordPolicy.reboot() to do this.

Since
KNOX 1.0

API used to write SELinux policy file to SEAndroid.

Returns
  • Returns POLICY_SET_OK if the policies were correctly set on device, POLICY_SET_REFUSED if the operation was rejected due to the lack of the ownership and POLICY_SET_FAILED if the policies was not set on device.
Throws
SecurityException If caller does not have required permissions
Usage
Used to copy the SELinux Policy File to the File System and trigger the load of the policy on SEAndroid. 

 SEAndroidPolicy mSEAndroidPolicy = SEAndroidPolicy.getInstance(mContext);
 
 int ret = SEAndroidPolicy.POLICY_SET_FAILED;
 
 try {
     ret = mSEAndroidPolicy.setSELinuxPolicy(sePolicy);
 } catch (SecurityException e) {
     Log.w(TAG, "SecurityException: " + e);
 }
Permission
The use of this API requires the caller to have the "android.permission.sec.MDM_SEANDROID" permission which has a protection level of signature.
