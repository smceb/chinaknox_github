public boolean activateContainer ()

Since: API Level KNOX_1.0

This method is deprecated.
Since KNOX 2.0

API to activate container.

Returns
  • true if container is activated, else false.
Throws
SecurityException If caller does not have required permissions. Only administrator who created container can enforce policy, if the calling administrator is not an owner of container the API throws SecurityException.
IllegalArgumentException If container does not exist or container creation/removal is in progress.
Usage
This API can be used by device administrator to activate container after pushing basic MDM policies. 
 // When you create container successfully, containerID will be returned via intent.
 // Use this containerID in below API.
 EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
 try {
     EnterpriseContainerManager containerMgr = ekm.getEnterpriseContainerManager(containerID);
     boolean success = containerMgr.activateContainer();
 } catch (SecurityException e) {
     Log.w(TAG, "SecurityException: " + e);
 } catch (IllegalArgumentException e) {
     Log.w(TAG, "IllegalArgumentException: " + e);
 }

Permission
The use of this API requires the caller to have the "android.permission.sec.MDM_ENTERPRISE_CONTAINER" permission which has a protection level of signature.

public static boolean createContainer (EnterpriseContainerCallback callback, int flag)

Since: API Level KNOX_1.0

This method is deprecated.
Use createContainer(String)

API to create a new Enterprise application container.

Parameters
callback Callback to be called when know if container was created successfully or it failed. Callback object is of type EnterpriseContainerCallback.
flag Flag to apply feature in create container flow FLAG_DISABLE_CONTAINER_ACTIVATION_FLOW , FLAG_ENABLE_CONTAINER_ACTIVATION_FLOW .
Returns
  • True if container is being created. False is returned if something in the starting process for the container's creation fails. Use the specified EnterpriseContainerCallback to make sure the container creation was successful
Throws
SecurityException If caller does not have required permissions.
Usage
Use this API to create Enterprise Application container. You need to specify a callback to be notified if the creation was successful or not. This API will return immediately but the container creation will take some time to finish, depending on the number of applications to be installed. 

 int flag = FLAG_ENABLE_CONTAINER_ACTIVATION_FLOW;
 boolean status = EnterpriseContainerManager.createContainer(new ApplicationContainerCallback(),
         flag);
 // ApplicationContainerCallback extends EnterpriseContainerCallback

Permission
The use of this API requires the caller to have the "android.permission.sec.MDM_ENTERPRISE_CONTAINER" permission which has a protection level of signature.

Since

public boolean isContactInfoToNonKnoxAllowed ()

Since: API Level KNOX_1.0.2

This method is deprecated.
Since KNOX 2.0

API to check whether the policy to share the contact information to outside of container is enforced.

Returns
  • true if the policy is enforced or false otherwise.
Throws
IllegalArgumentException If container does not exist or container creation/removal is in progress.
Usage

 // When you create container successfully, containerID will be returned via intent.
 // Use this containerID in below API.
 EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
 try {
     EnterpriseContainerManager containerMgr = ekm.getEnterpriseContainerManager(containerID);
     boolean isAllowed = containerMgr.isContactInfoToNonKnoxAllowed();
     if (true == isAllowed) {
          // policy to share the contact information is enforced
     } else {
          // policy to share the contact information is not enforced
     }
 } catch (IllegalArgumentException e) {
     Log.w(TAG, "IllegalArgumentException: " + e);
 }

public boolean lock ()

Since: API Level KNOX_1.0

This method is deprecated.
Use lock()

API to lock the container.

Returns
  • true if the container is successfully locked.
Throws
SecurityException If caller does not have required permissions. Only administrator who created container can enforce policy, if the calling administrator is not an owner of container the API throws SecurityException.
IllegalArgumentException If container does not exist or container creation/removal is in progress.
Usage
Use this API to lock the container. It will return true if the container was locked successfully. Calling this API will chage the status of the container to CONTAINER_LOCKED 

 // When you create container successfully, containerID will be returned via intent.
 // Use this containerID in below API.
 EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
 try {
     EnterpriseContainerManager containerMgr = ekm.getEnterpriseContainerManager(containerID);
     boolean success = containerMgr.lock();
 } catch (SecurityException e) {
     Log.w(TAG, "SecurityException: " + e);
 } catch (IllegalArgumentException e) {
     Log.w(TAG, "IllegalArgumentException: " + e);
 }

Permission
The use of this API requires the caller to have the "android.permission.sec.MDM_ENTERPRISE_CONTAINER" permission which has a protection level of signature.

public static boolean removeContainer (int containerId, EnterpriseContainerCallback callback)

Since: API Level KNOX_1.0

This method is deprecated.
Use removeContainer(int)

API to remove Enterprise application container. You need to specify a callback to be notified if the removal was successful or not. This API will return immediately but the container removal will take some time to finish, depending on the number of applications to be uninstalled.

Parameters
containerId The ID of the container to be deleted. If container does not exist, this API has no effect.
callback Callback to be called when know if container was created successfully or it failed. This object is of type EnterpriseContainerCallback.
Returns
  • True if container is being removed. False is returned if something in the starting process for the container's removal fails. Use the specified EnterpriseContainerCallback to make sure the container removal was successful
Throws
SecurityException If caller does not have required permissions. Only administrator who created container can enforce policy, if the calling administrator is not an owner of container the API throws SecurityException.
IllegalArgumentException If container does not exist or container creation/removal is in progress.
Usage
This API can be used to remove an existing container. If container does not exist, this API has no effect. 

 boolean status = EnterpriseContainerManager.removeContainer(containerId,
         new ApplicationContainerCallback());
 // ApplicationContainerCallback extends EnterpriseContainerCallback

Permission
The use of this API requires the caller to have the "android.permission.sec.MDM_ENTERPRISE_CONTAINER" permission which has a protection level of signature.

public boolean setPasswordEnabledContainerLockTimeout (long timeMs)

Since: API Level KNOX_1.0.2

This method is deprecated.
Since KNOX 2.0

API to enforce the policy to lock the container with timeout and the user will be able to unlock the container with the password. An administrator can remove this policy by invoking the API with timeout as zero.

Returns
  • true if policy is applied successfully or false otherwise.
Throws
SecurityException If caller does not have required permissions. Only administrator who created container can enforce policy, if the calling administrator is not an owner of container the API throws SecurityException.
IllegalArgumentException If container does not exist or container creation/removal is in progress.
Usage

 // When you create container successfully, containerID will be returned via intent.
 // Use this containerID in below API.
 EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
 try {
     long timeout = 60000; // 60 seconds for example
     EnterpriseContainerManager containerMgr = ekm.getEnterpriseContainerManager(containerID);
     boolean success = containerMgr.setPasswordEnabledContainerLockTimeout(timeout);
     if (true == success) {
          // policy enforced successfully
     } else {
          // failed to enforce policy
     }
 } catch (SecurityException e) {
    Log.w(TAG, "SecurityException: " + e);
 } catch (IllegalArgumentException e) {
     Log.w(TAG, "IllegalArgumentException: " + e);
 }

Permission
The use of this API requires the caller to have the "android.permission.sec.MDM_ENTERPRISE_CONTAINER" permission which has a protection level of signature.

public boolean unlock ()

Since: API Level KNOX_1.0

This method is deprecated.
Use unlock()

API to unlock the container.

Returns
  • true if the container is successfully unlocked.
Throws
SecurityException If caller does not have required permissions. Only administrator who created container can enforce policy, if the calling administrator is not an owner of container the API throws SecurityException.
IllegalArgumentException If container does not exist or container creation/removal is in progress.
Usage
Calling this API will unlock the container. It will return true if the container was successfully unlocked. Calling this API will change the status of the container to CONTAINER_INACTIVE. The container is in the CONTAINER_INACTIVE state when administrator calls unlock after locking the container. 


 // When you create container successfully, containerID will be returned via intent.

 EnterpriseKnoxManager ekm = EnterpriseKnoxManager.getInstance();
 try {
     EnterpriseContainerManager containerMgr = ekm.getEnterpriseContainerManager(containerID);
     boolean success = containerMgr.unlock();
 } catch (SecurityException e) {
     Log.w(TAG, "SecurityException: " + e);
 } catch (IllegalArgumentException e) {
     Log.w(TAG, "IllegalArgumentException: " + e);
 }

Permission
The use of this API requires the caller to have the "android.permission.sec.MDM_ENTERPRISE_CONTAINER" permission which has a protection level of signature.
